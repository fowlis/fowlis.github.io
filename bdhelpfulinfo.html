Installer Issues

Installer does not open

Solutions:
  • Redownload installer
  • 7zip method:
    • Install 7-Zip or similar software.
    • Right click the "BetterDiscord-windows.exe", from the 7-Zip options, click extract to "BetterDiscord-Windows\"
    • Open the new "BetterDiscord-Windows" folder and run the "BetterDiscord.exe"
  • Manual install
Related issue: "Error opening file for writing"
Press "Ignore" several times, or try the above solutions.

Discord missing in installer / BD settings missing after install

The user should reinstall Discord just by installing it again. If this doesn't work, they should be prompted to fully reinstall Discord.
Windows guide
MacOS guide
For BD settings missing, make sure the user actually installs BD again after reinstalling Discord.

Linux users:
BD not appearing in the installer usually means that the user has not installed Discord the normal way via discord.com
The user should use BetterDiscordCTL, as this supports Appstore/Flatpak/Snap/whatever installs.


"Cannot read property 'assets' of undefined"

User should redownload the installer, the version they are using is extremely old.

Windows only - EPERM: operation not permitted, open


The installer does not have the correct permissions for installing, can be resolved with a complete reinstall of Discord.
Guide

The installer does not have the correct permissions for installing, grant the installer full disk access

General Issues

BetterDiscord disappearing after Discord/PC restart

Stable - prompt the user to do a full reinstall of Discord, as their client is probably corrupted in some way and is forcing updates on every restart.
PTB/Canary - explain to the user that they should either move to Stable, or deal with it. These clients receive updates much more often than Stable and as such the issue will be more prevalent.
A solution to this is apparently in the works for BD, but nothing yet.

You can also offer Knew's quick reinstall scripts to make the process of reinstalling BD easier and quicker.

User's installed addons not showing in addon tabs

There are a couple possible causes for this:
  • File shows as "download.html". The user has a download manager, and it is downloading the wrong file. Prompt the user to disable/remove their download manager for the file to download correctly.
  • The filename looks almost correct, such as "ZeresPluginLibrary plugin.js". Instruct the user to add the missing "." between the addon's name and the "plugin"/"theme" in the filename.

Addons giving errors immediately upon loading them

  • "META not found". Ask the user to either screenshot the contents of the file, or send the file. Check that the file actually contains the appropriate code for the addon. Redownloading the file should fix it, but you can also direct the user to copy-paste the proper code into the file in case redownloading doesn't work.
  • Most other errors should be reported directly to the developer of the addon, you can either link their GitHub issues or direct the user to the author's support server if they have one.

BD logo flashing in bottom corner of games (PTB/Canary)

Prompt user to disable in-game overlay, or enable legacy overlay.
This should be fixed in the next update of BD.

Window snapping broken with BD

Caused by transparency and is a known bug with Electron. Not a BD issue to fix.

Plugin-related

Plugins breaking after a Discord/BD update

There is nothing we can do specifically to help the user, unless the issue is crashing or causing the user to not be able to disable their plugin(s).
If the user is experiencing crashing immediately after starting Discord as a result of a Discord/BD update, instruct them to do a Repair with the BD installer.
If the user knows what plugin is causing their issue, they should report it directly to the developer.
If the user does not know which plugin is causing their issue, we cannot help them much other than instruct them to do a binary search of their plugins folder to find the cause.
Then once they find the affected plugin(s), they can report the issue to the dev(s).

General issues with plugins

Again, we cannot help the user much if they know which plugin is causing the issue.
It may be possible to accurately identify what type of plugin could be causing it, i.e. an issue with the members tab could be caused by a plugin that interacts with it, such as MemberCount.
Knowledge like this could help the user find the afflicted plugin.

Plugins folder location

Some issues may cause the user to not be able to access their settings to open the plugins folder, so you can instruct them to open it manually.
  • Windows
    • Press Win + R and paste in %appdata%/betterdiscord/plugins
  • MacOS
    • Go to Finder, press Cmd + Shift + G and paste in ~/Library/Application Support/betterdiscord/plugins
  • Linux
    • Open terminal and paste in cd ~/.config/BetterDiscord/plugins, then xdg-open .

Theme-related

Theme breaks to the point of making it impossible to disable it

Doing a repair with the installer will disable all addons, so the user can remove the broken theme.
Alternatively, you can instruct them to use this snippet:
BdApi.Themes.getAll().forEach((theme) => BdApi.Themes.disable(theme)); document.getElementById("customcss").innerHTML = "";

Theme has broken elements, or is broken in general

The user should report this directly to the dev of the theme(s), if not done so already.
If there is a solution available that the dev has not merged, feel free to offer it to the user.

Themes folder location

Some issues may cause the user to not be able to access their settings to open the themes folder, so you can instruct them to open it manually.
  • Windows
    • Press Win + R and paste in %appdata%/betterdiscord/themes
  • MacOS
    • Go to Finder, press Cmd + Shift + G and paste in ~/Library/Application Support/betterdiscord/themes
  • Linux
    • Open terminal and paste in cd ~/.config/BetterDiscord/themes, then xdg-open .
It may be necessary to guide the user to the custom.css file in some cases, here are the paths to that:
  • Windows: %appdata%/betterdiscord/data/stable/custom.css
  • MacOS: ~/Library/Application Support/betterdiscord/data/stable/custom.css
  • Windows: ~/.config/BetterDiscord/data/stable/custom.css
These paths assume the user is on the Stable client, so if they are on PTB/Canary they should replace "stable" in those paths with their respective client name.
